# PR submission guide

## guidelines
- follow everything described in https://sgl-project.github.io/developer_guide/contribution_guide.html#
- scratch/ folder is for local testing myself and should not be pushed
- follow guideline to create tests in sglang format 
- make sure codebase exactly follows sglang style based on guidelines
- make sure codebase do not conflict with most updated main branch in sglang
- create PR ticket format described in sglang


## goal
ultimate goal is to create a feature branch with correct coding style, PR ticket conforming to standards showing bench results and visuals, benchmarking results, and a feature branch that is compatible with sglang main branch.
